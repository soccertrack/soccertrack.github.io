Runtime

I am a believer of Node and I like Ecmascript v6 (ES6). Node has strong library support. Thus coding my bot with this tech stack.

Crawl

These days, crawling is a little complicated, gone are the days that there’s some JSON API for you crawler to call into. Modern pages like to mingle data into Javascript and thereby harder to get with just a REST call.

The easier way to get these data is via browser DOM (document object model). It is easiest because DOM is the last place where final content will be organized and then rendered in the browser. There are many ways to handle this but I started with chrome extensions. This makes it possible for my bot to do action in the browser like clicking a button, scroll etc.

After having a version that works. I have a problem, chrome extensions require the browser to be activated and focus. This makes it impossible for me to run things in the background. I don’t really want to have a desktop tie to bot work. I need something that I can run in the background and simple REST calls won’t cut it. I did research and found something like JSDOM. I can probably use JSDOM but it doesn’t run script by default and I have to fetch all the content down. Furthermore running scripts in a local context can be dangerous. Something similar but better is Phantom JS. But they are losing developer support.

Finally I stumbled on Puppeteer from Google. It is pretty much what I needed. It is headless which means the browser instance doesn’t have to show a screen (headless), take screenshots and it provides control and interaction to web pages that I need. The following is a sample code to go to youtube, search for something, click and take a screenshot. Pretty trivial.

const browser = await puppeteer.launch()

   const page = await browser.newPage()

   await page.goto('https://youtube.com')

   await page.type('#search', 'latest videos')

   await page.click('button#search-icon-legacy')

   await page.waitForSelector('ytd-thumbnail.ytd-video-renderer')

   await page.screenshot({ path: 'youtube_list.png' })

Another advantage is Puppeteer actually uses the same chrome browser engine. Maintaining all cookies etc, just without the UI. Additional tip about crawling is to make sure to have good pause between crawls to overcome rate limiting on API gateway. Plus make sure you change your User Agent string to a regular browser.

The Bot

I need something that is lightweight and uses very little power. The obvious choice is a Raspberry Pi running Ubuntu 64. I got this Pi model from Amazon for $80+. Loaded in the minimum core Ubuntu 64 because it is a common OS sku for development. The website hosting

Github is a good deal for small teams (up to 5 users). It is free. It comes with issue tracker, source management, pages, kanban board .. you name it.. All free. Don’t know why but it is good for me.

One of the features I am leveraging is Github Pages. You push web pages into GitHub repository and Github will host those pages for you. CDN, multi-data center and high performance. New commits into the main branch take about 10-15 seconds to reflect on the live website.

You can host it with https://<projectname>.github.io or custom DNS. For that I spent $1/year for both domain names (https://ussoccertrack.com) and DNS. I did it through ionos.com. After getting access to DNS entries, I just have to map some DNS A and CNAME records to github’s IP and be done. To summarize, I am paying $1/year for DNS and domain name and $0 for hosting on a high performance site.

Features of ussoccertrack.com

The site is tracking 70-80 players playing in many countries. It is always sorted by recent appearances. Users can scroll or filter players by their position. For each player, you can get the player's bio, team, recent matches and game form for the last 5 games.

The same site is designed for both mobile and desktop. For example, you will see a single column view on a mobile phone and on your desktop you can see many columns based on how big the browser window is.

Clicking at any player, you will find more information about that player. Including player’s tweet, recent news, league standing, player overall stats, video highlights of recent games.

Remember since there’s no cost and no backend code for the project, I have to be creative to solve the sharing problem.

For the sharing to Facebook and Twitter, I am generating image data from my DOM. This can be done via a library HTML2Canvas. Once it is in HTML5 canvas, you can save it to image-data. The challenge came as FB or Twitter don’t take in image-data in the source attribute, only URL. I then created a free imgur developer account and set up for anonymous calls.
